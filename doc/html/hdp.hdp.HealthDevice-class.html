Class HealthDevice

source code

object --+
         |
        HealthDevice

The HealthDevice represents a single (endpoint, role, data_type) in the remote device.

Each device is bound to a given HealthApplication object, so data_type and role are implicitly defined (being the remote device's role the opposite of our application's).

The MDEP ID is also known by this class, if the remote device publishes a SDP record. If not, or if this device is created automatically when we are acceptors, MDEP ID is zero.
